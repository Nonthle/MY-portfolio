Cybercrime Dynamics System

Click to view details

Problem

Cybercrime incidents are rapidly increasing across digital networks, and traditional security systems struggle to model how attacks spread and evolve over time. Most systems are reactive rather than predictive, making it difficult to understand how vulnerabilities propagate within a network.

Approach

This project applies the SIR (Susceptible–Infected–Recovered) epidemiological model to simulate the spread of cyber threats within a network environment. In this context, users in a system are categorized into three states: Susceptible (S), representing vulnerable systems; Infected (I), representing compromised systems affected by cyber attacks such as malware or phishing; and Recovered (R), representing systems that have been secured or restored through mitigation strategies.

The model is extended using machine learning techniques to improve predictive capability. A Decision Tree classifier is used to identify risk patterns, Physics-Informed Neural Networks (PINNs) are used to model dynamic system behavior, and Principal Component Analysis (PCA) is applied for dimensionality reduction and feature extraction from cyber risk data.

Result

The integrated SIR and machine learning framework successfully captured the dynamic propagation of cyber threats within a network environment. The system identified critical transition patterns between susceptible, infected, and recovered states, enabling early detection of potential cyber attacks. The hybrid model improved prediction accuracy by approximately 30%, demonstrating the effectiveness of combining mathematical epidemiological models with data-driven machine learning techniques for cybersecurity analysis.

Models Used

  • Physics-Informed Neural Networks (PINNs)
  • Decision Tree
  • PCA

Packet Tracer Network Design

Click to view details

Problem

Basic network communication between devices often fails without proper configuration of switches, routers, and interfaces. Many small network setups lack proper structure, leading to connectivity issues and inefficient data flow.

Approach

A simple network topology was designed and implemented using Cisco Packet Tracer. The network consists of PCs connected through a switch and a router to enable communication between different segments.

Switch configuration was used to manage local connections, while the router was configured to handle communication between networks using appropriate interfaces (FastEthernet and GigabitEthernet ports). Proper IP addressing and interface activation were applied to ensure connectivity.

Result

The network was successfully configured, allowing communication between devices across the network. The setup demonstrates a clear understanding of basic networking concepts including device connectivity, interface configuration, and data flow between network components.
